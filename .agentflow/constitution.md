# agentflow-mvp AI Coding Constitution

## Purpose

This file defines the project rules every AI coding assistant must follow.
Keep it short, concrete, and current.

## Operating Rules

- Read `.agentflow/state.yaml` before starting project work.
- Read `.agentflow/skills/SKILL.md` before choosing a workflow.
- Do not start implementation until the current task has clear scope and acceptance.
- Prefer small changes with fresh verification evidence.
- Do not modify unrelated files.
- Record unresolved risks in the current change folder or final handoff.

## Project Boundaries

- Define what this project is responsible for.
- Define what this project deliberately does not do.
- Define directories or files that require extra caution.

## Verification Policy

- Every implementation handoff must list commands that were run.
- If verification was not run, state why and what remains risky.
- A task is not complete until acceptance evidence is written down.
